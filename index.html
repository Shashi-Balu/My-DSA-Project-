What is Disk Scheduling Algorithm?

Scheduling is mechanism of arranging requests in order to save un-necessary rotations by disk arm and disk pointer. Operating System takes I/O requests from queue and process them one by one. Algorithm used to select which I/O request is going to be selected first is called Disk Scheduling Algorithm. Objective of using Disk Scheduling Algorithm is:

  • To minimize the response time.
  • To maximize the throughput.

Structure of Disk

  • Disk is divided into tracks.
  • Each track is further divided into sectors.
  • Track is a circular path on the surface of a disk or diskette on which information is magnetically recorded and from which recorded information is read.
  • A platter is a circular magnetic plate that is used for storing data in a hard disk. A hard disk drive contains several platters that are mounted on the same spindle
  • Sector is one of the "pie slices" the disk is divided into. Dividing the circular medium into pie slices is a way to organize it so that data can be located by the read/write heads of the drive.
  • Actuator is a device which moves head arm assembly. Its task is, not only to start and secure continuous movement of hard drive head assembly, but also to fully control it.




Seek Time

Time taken to locate the disk arm to a specified track where the data is to be read or write. Disk scheduling algorithm that gives minimum average seek time is better.

Transfer Time

Time to transfer the data. It depends on the rotating speed of the disk and number of bytes to be transferred.

Disk Access Time

Disk Access Time = Seek Time + Rotational Latency + Transfer Time

Disk Response Time

Average of time spent by a request waiting to perform its I/O operation. Average Response time is the response time of the all requests.

Rotational Latency

Time taken by the desired sector of disk to rotate into a position so that it can access the read/write heads.

Type of Disk Scheduling Algorithm

FCFS - First Come First Serve

FCFS is the simplest of all the Disk Scheduling Algorithms. In FCFS, the requests are addressed in the order they arrive in the disk queue.

Advantages
  • Every request gets a fair chance
  • No indefinite postponement
Disadvantages
  • Does not try to optimize seek time
  • May not provide the best possible service

SSTF - Shortest Seek Time First

In SSTF requests having shortest seek time are executed first. The request near the disk arm will get executed first. SSTF is certainly an improvement over FCFS as it decreases the average response time and increases the throughput of system.

Advantages
  • Average Response Time decreases
  • Throughput increases
Disadvantages
  • Overhead to calculate seek time in advance
  • Can cause Starvation for a request.

SCAN

Also known as Elevator Algorithm the disk arm moves into a particular direction and services the requests coming in its path and after reaching the end of disk, it reverses its direction and again services the request arriving in its path.

Advantages
  • High throughput
  • Low variance of response time
  • Average response time
Disadvantages
  • Long waiting time for requests for locations just visited by disk arm

C-SCAN - Circular Scan

In CSCAN algorithm the disk arm moves to extreme end of the path while fulfiling the requests, and then come back to exteme end of other direction without fulfiling request and then move forward again to fulfil requests.

Advantages
  • Provides more uniform wait time compared to SCAN
Disadvantages
  • More seek movements as compared to SCAN Algorithm

LOOK

Disk does not go to the end of the disk goes only to the last request to be serviced in front of the head and then reverses its direction from there only. Thus it prevents the extra delay which occurred due to unnecessary traversal to the end of the disk.

Advantages
  • It provides better performance in comparison to the SCAN algorithm.
  • The LOOK scheduling algorithm avoids starvation.
Disadvantages
  • There is an overhead of finding the final requests.

C-LOOK - Circular Look

In CLOOK, the disk arm in spite of going to the end goes only to the last request to be serviced in front of the head and then from there goes to the other end's last request. Thus, it also prevents the extra delay which occurred due to unnecessary traversal to the end of the disk.

Advantages
  • It provides better performance compared to the LOOK disk scheduling algorithm.
  • The starvation is avoided in the C-LOOK disk scheduling algorithm.
  • There is minimal waiting time .
Disadvantages
  • The overhead of finding the end requests is present in C-LOOK.
